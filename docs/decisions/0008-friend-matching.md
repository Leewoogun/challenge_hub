# ADR-0008: 전화번호 등록 플로우 (SMS 인증 없이)

- **상태**: pending
- **생성**: 2026-04-23
- **영향 범위**: 온보딩, 설정, 친구 매칭 활성화 조건, `users.phone_verified` 의미

## 맥락

2026-04-23 기획 변경: **로그인 단계의 SMS 인증만 제거** (카카오 로그인만으로 식별). 친구 매칭은 여전히 전화번호부 해시 기반으로 동작하므로, 사용자 전화번호를 시스템에 등록하는 경로는 필요하다.

SMS 코드 인증 없이 전화번호를 어떻게 등록할지 결정이 필요. 아울러 기존 `users.phone_verified` 컬럼의 의미도 재정의해야 함.

## 선택지

### A. Kakao SDK OAuth scope로 획득 (권장)

- 카카오 로그인 scope에 `account_phone_number` 추가 요청. 사용자 동의 시 카카오 계정의 전화번호 자동 취득 (카카오가 이미 SMS 인증해 둔 번호).
- **장점**:
  - 사용자 타이핑 불필요
  - 카카오가 이미 SMS 인증한 번호라 정확도 보장
  - SMS 인프라 자체가 불필요
  - `phone_verified=true`의 근거로 사용 가능 (카카오 인증 경로)
- **단점**:
  - 카카오디벨로퍼 비즈니스 채널 등록·검수 필요할 수 있음
  - 전화번호를 미등록한 카카오 계정에는 적용 불가 → fallback 필요

### B. 사용자 직접 입력 (검증 없음)

- 온보딩 또는 "연락처로 친구 찾기" 진입 시 전화번호 입력.
- **장점**: 구현 가장 단순, 카카오 권한 검수 불필요
- **단점**:
  - 타인 전화번호 입력 가능 (도용 시 해당 번호를 가진 진짜 유저의 친구 매칭에 영향)
  - `phone_verified`는 항상 false → 컬럼 의미 재검토 필요

### C. 지연 등록 (Lazy)

- 기본은 미등록. 사용자가 "연락처로 친구 찾기"를 눌렀을 때 비로소 전화번호 입력·등록 스텝 진입.
- **장점**:
  - 개인정보 최소 수집 원칙에 부합
  - 전화번호 없이도 앱 핵심 기능(닉네임 검색으로 친구 추가, 챌린지) 사용 가능
- **단점**: 자동 친구 발견 기회 감소

### D. A + C 혼합 (권장)

- 카카오 로그인 시 `account_phone_number` scope를 **선택 동의**로 요청 → 동의하면 자동 등록, 거부하면 미등록.
- 나중에 "연락처로 친구 찾기" 진입 시 다시 유도 (지연 등록).
- 최종 fallback으로 B(직접 입력) 옵션도 제공.

## 트레이드오프

| 축 | A | B | C | **D (A+C)** |
|----|---|---|---|-------|
| 구현 난이도 | 중 (Kakao scope 검수) | 저 | 저 | 중 |
| 검증 정확도 | 높음 (카카오 인증분) | 없음 | 해당 흐름에 따라 다름 | 높음 (A 경로 시) |
| 사용자 편의 | 높음 | 중 | 중 | 높음 |
| 개인정보 최소화 | 중 | 중 | **높음** | 높음 |
| 친구 자동 매칭 커버리지 | A 동의 비율만큼 | 입력자만 | 지연 등록자만 | 최대 |

## 권장

**D (A + C 혼합).** 기본 경로는 카카오 scope 자동 등록, 거부자에겐 지연 등록 유도. B(직접 입력)는 최종 fallback.

## `phone_verified` 컬럼 재정의 (하위 결정)

SMS 인증이 사라진 이상 이 컬럼의 의미가 모호. 세 가지 옵션:

1. **삭제**: 단순함. 하지만 향후 "검증된 번호 vs 미검증 번호" 구분이 필요할 수 있어 성급할 수 있음.
2. **유지 + 의미 재정의**: `phone_verified=true` = "카카오가 인증한 번호"(선택지 A 경로). 직접 입력(B)은 false. (권장)
3. **이름 변경**: `phone_source VARCHAR(20)` — 값: `kakao_oauth` / `user_input` / `null`. 더 명확하지만 마이그레이션 필요.

권장은 **2**: 현재 스키마 유지, 의미만 재정의하여 문서화.

## 실행 체크리스트 (결정 시)

- [ ] 카카오디벨로퍼 `account_phone_number` scope 요청 가능 여부 확인
- [ ] 비즈니스 채널 등록·검수 비용·시간 파악
- [ ] 검수 실패 시 B 경로로 전환 가능한 구조
- [ ] `phone_verified` 의미 재정의 결정 (위 세 옵션 중 선택)
- [ ] 전화번호 도용·오타 방지 대안 필요 여부 (차단/신고 기능과 연결)
- [ ] `users.phone_number`는 SHA-256 해시 저장 유지 (기존 정책, Server 6.2)
- [ ] 온보딩에서 설명 문구: "연락처로 친구 찾기에 사용됩니다, 건너뛸 수 있습니다"
