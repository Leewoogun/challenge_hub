# ADR-0008: 친구 매칭 방식 (전화번호부 연동 대체)

- **상태**: pending
- **생성**: 2026-04-23
- **영향 범위**: 친구 검색·추가 UX, Server Friends API 설계, 온보딩 플로우

## 맥락

2026-04-23 기획서 변경: **로그인 단계에서 전화번호 인증 제거** (카카오 로그인만). 이에 따라 전화번호부 기반 친구 매칭(`POST /api/v1/friends/sync-contacts`)도 함께 제거됨.

친구를 찾는 대안 메커니즘이 필요하다.

## 선택지

### A. 닉네임/ID 검색만

- `GET /api/v1/friends/search?query=<nickname>` 로 텍스트 검색.
- **장점**: 구현 단순, 추가 인프라 없음.
- **단점**: 이름 중복 다수일 때 식별 어려움. 본인이 누군지 정확히 알려줘야 함(스크린샷 공유 등).
- **보완**: 닉네임#해시태그(예: `길동#0420`) 방식으로 중복 해소.

### B. Kakao 친구 목록 연동

- Kakao SDK의 친구 목록 API로 카카오 친구 중 앱 가입자 제안.
- **장점**: 소셜 그래프 즉시 활용, 매끄러운 온보딩.
- **단점**: 카카오 비즈니스 채널 추가 권한 검수 필요 가능성. 카카오 친구가 아닌 사람은 못 찾음.

### C. 초대 코드 (6~8자)

- 유저마다 고유 코드. 코드 입력으로 친구 요청.
- **장점**: 오프라인에서 코드만 교환하면 됨. 구현 간단.
- **단점**: 코드 타이핑 UX 불편. 공유 채널은 여전히 외부(카톡/문자).

### D. 초대 링크 (딥링크)

- `https://maengse.app/invite/<token>` 링크 공유 → 상대가 탭하면 앱에서 즉시 친구 요청.
- **장점**: 공유 즉시성, 카카오톡 공유가 국내 사용자에게 친숙.
- **단점**: 딥링크 설정(Android App Links, iOS Universal Links), 토큰 만료·재사용 정책 필요.

### E. A + D 혼합 (권장)

- **기본**: 닉네임#해시태그 검색 (A).
- **주력 온보딩**: 초대 링크 공유 (D) — 카톡 공유가 국내 사용자에게 자연스러움.
- B는 Kakao API 검수 통과 후 옵션으로 확장.

## 권장

**E (A + D 혼합)** — MVP에서는 닉네임#해시 + 초대 링크로 시작. B(Kakao 친구 API)는 Kakao 검수 완료 후 점진 도입.

## 실행 체크리스트 (결정 시)

- [ ] 닉네임 유일성 규칙: 중복 허용 + `#4자리 해시` 자동 생성 (예: `길동#0420`)
- [ ] 초대 링크 도메인 확보 (예: `maengse.app` 또는 `challenge.app`)
- [ ] Android App Links(assetlinks.json), iOS Universal Links(apple-app-site-association) 설정
- [ ] 초대 토큰 만료 정책: 1회성 vs 7일 유효
- [ ] Server API 신설·수정:
  - `GET /api/v1/friends/search?query=` — 닉네임·해시·초대코드 통합 검색
  - `POST /api/v1/friends/invite` — 초대 토큰 발급
  - `POST /api/v1/friends/invite/{token}/accept` — 수락
- [ ] DB `users` 테이블에 `nickname_hash VARCHAR(8)` 컬럼 추가 (유일성 보장을 위한 해시 부분)
- [ ] Rate limiting: 초대 발급 일당 20회 등
- [ ] `docs/features/friend-matching/` 로 sub-feature 분리
